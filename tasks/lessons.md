# Lessons

Формат записи: что случилось → почему → как не повторить.

## 1. Redis event loop crash при втором запросе
Что: `pipeline_lock.py` падал на второй запрос.
Почему: не трекалась ссылка на event loop между вызовами.
Как не повторить: явно трекать ссылку на event loop, не пересоздавать его неявно.

## 2. ModuleNotFoundError
Что: модуль не находился при запуске.
Почему: запуск был прямым скриптом, а не через -m.
Как не повторить: всегда `python -m ubt_os.main`, никогда напрямую.

## 3. Отсутствовали таблицы video_analytics и knowledge_entries
Что: код писал в таблицы, которых не было в Supabase.
Почему: SQL-схемы не были применены полностью при первом деплое.
Как не повторить: чек-лист всех таблиц перед стартом сервиса, сверка со схемой.

## 4. JSON parsing fail на markdown fences
Что: `json.loads()` падал, когда Claude оборачивал ответ в \`\`\`json.
Почему: ответ модели не очищался от markdown-обёртки перед парсингом.
Как не повторить: всегда прогонять ответ через `_extract_json()` regex-хелпер перед `json.loads()`.

## 5. _run_obsidian был алиасом на git sync
Что: ожидалась запись файлов в vault, а происходил git sync.
Почему: метод был по ошибке алиасом не на то, что нужно.
Как не повторить: явно тестировать побочный эффект метода, не только то, что он не падает.

## 6. Mismatch AttributionTracker vs AttributionAnalyzer
Что: импорт класса падал по имени.
Почему: класс переименовали в одном месте, не везде.
Как не повторить: после переименования класса — grep по всему репозиторию на старое имя.

## 7. IndexError на пустом platform_priority
Что: `strategy_engine.py` падал на пустом списке.
Почему: не было проверки на пустой список перед индексацией.
Как не повторить: всегда проверять пустые коллекции перед прямой индексацией, особенно в логике с внешними данными.

## 8. Дублирующийся key conflict на daily_queues
Что: вставка дублирующегося ключа роняла операцию.
Почему: использовался plain INSERT вместо upsert.
Как не повторить: для таблиц с потенциальными повторами — upsert по умолчанию, не INSERT.

## 9. ThreadingHTTPServer не держал конкурентные запросы
Что: сервер не справлялся с параллельными запросами.
Почему: использовался однопоточный HTTP-сервер.
Как не повторить: для сервисов с конкурентной нагрузкой — сразу threading/async сервер, не однопоточный.

## 10. JSONDecodeError на пустом теле HTTP-запроса
Что: входящий запрос с пустым/нефинальным телом ронял json.loads() с трейсбеком (но сервис не падал).
Почему: обработчик не валидировал тело запроса перед парсингом.
Как не повторить: валидировать тело запроса до json.loads() — читать raw bytes, проверять raw.strip(), оборачивать в try/except JSONDecodeError с возвратом 400.
