# TikTok: Прогрев аккаунта (Account Warmup)

Связан с: [[zarubejka-geo-algorithms]] | [[tiktok-distribution-algorithm]] | [[affiliate-tools-directory]]

---

## Почему прогрев критичен

Первые **3–14 дней** аккаунта определяют траекторию навсегда:
- Без прогрева → плато 200–500 просмотров (участь большинства)
- С прогревом → первые ролики на 5K–50K просмотров (зелёный свет)

**Цель прогрева:** доказать TikTok два факта одновременно:
1. Ты = живой человек (не бот)
2. Ты = человек из целевой страны (US/BR/MX и т.д.)

## Инфраструктура (до создания аккаунта)

### Устройство
- Уникальный Android-телефон или cloud phone (GeeLark, Multilogin)
- Уникальный device fingerprint: GLOBAL или US IMEI
- **НЕЛЬЗЯ**: делить одно устройство между несколькими аккаунтами
- **Антидетект браузер** (Dolphin Anty) если через desktop — уникальный профиль

### Прокси
- **Только**: мобильный (4G/5G) или высококачественный residential proxy
- **НЕЛЬЗЯ**: VPN — TikTok детектирует VPN IP-диапазоны мгновенно → бан
- **НЕЛЬЗЯ**: датацентровые прокси (легко детектируются)
- Один прокси = один аккаунт (строго!)
- Для US: привязка к конкретному US-адресу (улица, город) — не просто US IP-пул

### SIM-карта / номер телефона
- Регистрация через номер целевой страны
- TikTok проверяет SIM страну при регистрации → это GEO-сигнал #2
- **Для US**: Airalo US eSIM ($5–15) — работает на iPhone и Android
- Альтернативы: Holafly, Maya Mobile
- **НЕЛЬЗЯ**: регистрация через RU номер для US-аккаунта

### GPS
- iOS: отключить "Allow Tracking" в настройках
- Android: отключить доступ к местоположению для TikTok
- Очистить кэш TikTok перед первым запуском

### Geo-consistency правило
**Всё должно указывать на одну страну:**
```
Device fingerprint (GLOBAL/US) + Proxy (US mobile) + SIM (US eSIM) + 
Регистрационный номер (US) + Язык системы (EN) = Аккаунт воспринимается как US
```

## День-за-днём план прогрева

### Для новых self-reg аккаунтов (14 дней)

| День | Лайки | Подписки | Комментарии | Публикации | Заметки |
|------|-------|----------|-------------|------------|---------|
| 1 | 0 | 0 | 0 | 0 | Только скроллинг FYP 20–30 мин |
| 2 | 15–20 | 0 | 0 | 0 | Лайки похожего контента |
| 3–4 | 30 | 5–10 | 1–2 | 0 | Начинать подписываться в нише |
| 5–6 | 40 | 10–15 | 3–5 | 1 нейтральное | Первое видео не по офферу |
| 7 | 40 | 15 | 5 | 1 тест | Тест видео близкое к теме оффера |
| 8–10 | 50 | 15–20 | 5–10 | 2–3/день | Нарастить активность |
| 11–14 | 50 | 20 | 10 | 2–3/день | 70% нейтральный / 30% целевой контент |

**Bio-ссылка**: добавлять только после 1,000 фолловеров ИЛИ не раньше дня 10–12.

### Для купленных аккаунтов с историей (5–7 дней)

| День | Активность |
|------|-----------|
| 1–2 | Скроллинг, лайки. Не менять IP, не менять настройки |
| 3–4 | Первые 1–2 публикации в нише аккаунта |
| 5–7 | Полная активность, переход к целевому контенту |

## Типичные ошибки при прогреве

| Ошибка | Последствие |
|--------|-------------|
| VPN вместо прокси | Моментальный бан или shadow ban |
| Одинаковый паттерн активности на всех аккаунтах | Детект фермы → массовый бан |
| Один и тот же нейтральный ролик на все аккаунты | Детект копирования → shadow ban |
| Ссылка в bio с первого дня | Алгоритм помечает как спам |
| Запуск всей фермы в один день | Детект паттерна. Запускать группами по 5–10 с разрывом 2–3 дня |
| RU proxy + US SIM | Конфликт сигналов = смешанный охват |
| Датацентровый прокси | Детект TikTok → низкий траст аккаунта |

## Ферма аккаунтов для UBT OS

### Структура фермы (рекомендуемая)

```
Нутра US (Tier-1):
  10 аккаунтов × GLOBAL device + US mobile proxy + Airalo US eSIM
  Dolphin Anty: отдельный профиль для каждого
  Прогрев: 14 дней → потом A26 BlatoPublisher

Нутра BR/MX (Tier-2 COD):
  10 аккаунтов × GLOBAL device + BR/MX proxy
  Прогрев: 7–10 дней (алгоритм менее строгий)
  
Беттинг образовательный:
  5 аккаунтов × аналогично нутра US
  Контент: спорт-аналитика, без слова "ставки"
```

### Масштабирование фермы
- Старт: группа 5 аккаунтов → оценить результаты → следующая группа
- Никогда не запускать все аккаунты одновременно
- Ротация прокси: меняй каждые 30 дней для снижения детекта

## Контент в период прогрева

### Нейтральный контент (дни 1–7)
- Общий lifestyle / развлечения / hacks в нише (без упоминания продукта)
- Пример для нутра: "5 morning habits that changed my life" (без упоминания добавок)
- Пример для беттинга: "How I analyze football matches" (без слова "ставки")

### Переходный контент (дни 8–14)
- 70% нейтральный + 30% нишевой с плавным упоминанием продукта
- По-прежнему: без прямого CTA, без ссылок в видео

### После прогрева (день 15+)
- Полный контент по офферу
- Bio ссылка → Keitaro → прелендинг → лендинг
- A26 BlatoPublisher берёт на себя постинг

## Применение для UBT OS

### Интеграция с агентами

```
Перед A26 BlatoPublisher добавить проверку:
  1. account_age_days >= 14 (для new self-reg)
  2. warmup_status == "completed"
  3. device_fingerprint in ["GLOBAL", "US"]
  4. proxy_type in ["mobile", "residential"]

Если не прошёл → вернуть "WARMUP_REQUIRED" статус
```

### Чеклист для каждого нового аккаунта

- [ ] Устройство: GLOBAL или US fingerprint
- [ ] Прокси: мобильный или residential US
- [ ] SIM: Airalo US eSIM для US-аккаунтов
- [ ] GPS: отключён в приложении
- [ ] Язык системы: English (US)
- [ ] Регистрация: через US номер
- [ ] Прогрев: 14 дней (new) / 7 дней (aged)
- [ ] Bio-ссылка: только после дня 10–12 и/или 1K фолловеров
- [ ] Dolphin Anty профиль: создан с уникальным fingerprint
