**GEO-таргетинг: 3 алгоритма для выхода на US-аудиторию**

Для показа контента на США нужно удовлетворить 3 алгоритма платформы:
1. Алгоритм ПРОИСХОЖДЕНИЯ — оценивает device ID (IMEI) ещё до любого контента
2. Алгоритм ПОВЕДЕНИЯ — ⏳ ожидает данных
3. Алгоритм ВЫДАЧИ КОНТЕНТА — ⏳ ожидает данных

**Алгоритм 1 — ORIGIN:**
- Платформа читает уникальный цифровой след устройства (IMEI/device fingerprint)
- Типы device ID: RU / CN / USA / GLOBAL
- RU или CN → охват режется на нулевом этапе
- GLOBAL → нейтральный идентификатор, не привязан к стране → лучший доступный старт
- "Правильный девайс" = ~20% успеха, но без него всё остальное не работает

**Применение для UBT OS:**
- Серверы/VPS с RU-IP и RU device fingerprint режут органику ещё до публикации
- При автопостинге через Blotato проверить: с каких устройств идёт активность аккаунтов
- Приоритет: использовать устройства/эмуляторы с GLOBAL или US device ID
- Источник упоминает поставщика "глобальных" устройств — уточнить на курсе

**Алгоритм ПОВЕДЕНИЯ: технический прогрев аккаунта**

- Маскировка IP с привязкой к конкретному US-адресу (не просто US-прокси)
- Billing address + платёжные реквизиты как у настоящего американца
- Human-behavior simulation (лайки, комменты, рекомендательная лента)

**Алгоритм 3 — ВЫДАЧА КОНТЕНТА (Distribution)**

Последний из трёх фильтров для US-аудитории. Собран из открытых источников (оригинал курса ждём).

**Механика тест-когорты:**
- После публикации TikTok показывает видео 200–500 пользователям за 1–3 часа
- Когорта формируется по GEO-сигналам аккаунта + контента
- Порог для продвижения в следующую волну: completion rate ≥70% (в 2025 подняли с 50%)

**GEO-сигналы (по приоритету):**
1. IP в момент публикации (самый сильный — определяет географию первой когорты)
2. SIM страна аккаунта (домашний регион)
3. Язык речи + on-screen текст (усиливает или размывает GEO)
4. Язык captions (SEO + вторичный сигнал)
5. Хэштеги (минимальны в 2026 — алгоритм читает контент напрямую)

Конфликт сигналов (US IP + не-US SIM) = смешанный охват, НЕ US.

**Ключевые метрики для дальнейшей дистрибуции:**
- Completion rate ≥70% = главный порог виральности
- Saves (>2%) и Shares (>1%) весят больше лайков
- Rewatch rate = сильнейший сигнал качества
- Comments > Likes в текущем алгоритме

**Технология контента для US:**
- Первые 3 секунды: EN текст на экране + pattern interrupt (цифра/шок/вопрос)
- Структура 60 сек: хук → проблема → трансформация → climax → мягкий CTA
- Caption на английском, 3–5 ключевых слов предложениями (не хэштегами)
- Трендовый US-аудио = хвост чужой US-аудитории

**Применение для UBT OS:**
- A21 content_creator → добавить EN on-screen text в сценарии → это GEO-сигнал
- A23 youtube_creator → retention-паузы каждые 15–20 сек для completion ≥70%
- A19 text_humanizer → AI-маркеры снижают естественность → снижается completion rate
- Цель: completion ≥70%, saves >2%, rewatch >0%
