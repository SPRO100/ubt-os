# Алгоритмы GEO-таргетинга для выхода на US-аудиторию

## Контекст
Материал из курса "Зарубежка" — практика работы с иностранной аудиторией (фокус на США).
Применяется к вертикалям **betting / nutra** на GEO **US, BR, MX, DE, PL**.

---

## 3 алгоритма платформы

Для показа контента на целевое GEO нужно пройти все три фильтра:

| # | Алгоритм | Статус |
|---|---|---|
| 1 | ПРОИСХОЖДЕНИЯ (Origin) | ✅ Разобран |
| 2 | ПОВЕДЕНИЯ (Behavior) | ✅ Разобран |

## Алгоритм 2 — BEHAVIOR (Поведение) / Технический прогрев

### Временное окно
Первые **3–4 дня** владения аккаунтом определяют всё:
- Плато 200–500 просмотров (участь большинства) — если прогрев не сделан
- Зелёный свет: первые ролики на 5–10К просмотров — если всё правильно

### Что такое технический прогрев
Цель — доказать алгоритму два факта одновременно:
1. Ты = живой человек (не бот, не робот, не мошенник)
2. Ты = человек из Америки (не смотря на реальную географию)

### Что входит в прогрев
**Верхушка айсберга (видимое):**
- Лайки, репосты, комментарии, настройка рекомендательной ленты

**Технический уровень (скрытое):**
- Маскировка IP-адресов + привязка к конкретному существующему US-адресу
- Получение billing-адреса и платёжных реквизитов (как у настоящего американца)
- Авторские тонкости (9-урочный модуль в курсе)

### Ключевой принцип
Нельзя начинать постить контент, пока не доказана "человечность" + US-география.
Контент до прогрева = рулетка.

### Применение для UBT OS
- **Blotato-автопостинг** = паттерн бота → нужен слой human-behavior simulation перед публикацией
- IP для аккаунтов: не просто US-прокси, а привязка к конкретному US-адресу (улица, город)
- Нужен billing address + платёжные данные для верификации US-личности аккаунта
- Warmup-период (3–4 дня) должен быть зашит в логику запуска каждого нового аккаунта
- В курсе есть ссылки на "секретные сервисы" для этого — уточнить

## Алгоритм 1 — ORIGIN (Происхождение)

### Как работает
Платформа считывает уникальный цифровой след устройства — аналог IMEI/device fingerprint.
Это происходит **до** любой оценки контента.

### Типы device ID
| ID | Что означает | Влияние |
|---|---|---|
| `RU` | Российское устройство | Охват режется на старте |
| `CN` | Китайское устройство | Охват режется на старте |
| `USA` | Американское устройство | Максимальное преимущество, сложнее добыть |
| `GLOBAL` | Нет привязки к стране | Нейтральный старт — оптимальный выбор |

### Ключевые цифры
- Правильный device = **~20% от итогового успеха**
- Без этих 20% остальные 80% не работают

### Рекомендация из курса
Выбор в пользу `GLOBAL` — нейтральный идентификатор, не требует "американского" устройства, доступен без поездок за рубеж.

## Применение для UBT OS

### Проблема
Серверы и устройства с `RU` device fingerprint режут органический охват ещё до публикации контента. При автопостинге через **Blotato** это означает, что аккаунты могут системно недополучать US/EU трафик.

### Что проверить
- [ ] С каких устройств/серверов идёт активность аккаунтов в Blotato
- [ ] Какой device fingerprint у используемых эмуляторов Android/iOS
- [ ] IP-адреса: нужны не просто US-прокси, а связка IP + device ID в одной "стране"

### Что делать
1. **Устройства:** использовать телефоны/эмуляторы с `GLOBAL` или `US` device ID
2. **Прокси:** мобильные US-прокси (не дата-центр — легко детектятся) в паре с нужным device ID
3. **Уточнить:** поставщик "глобальных" устройств упомянут в курсе — получить контакт

### Приоритет
`CRITICAL` — это фундамент. Без правильного device ID контент-стратегия не даст US-охвата.

## Алгоритм 3 — DISTRIBUTION (Выдача контента)

> Источник: синтез TokPortal, Hootsuite, VPNtoUS, PostEverywhere — данные 2025–2026
> Обновлено: 2026-06-29

### Как работает тест-когорта

После публикации TikTok показывает видео **200–500 случайным пользователям** за первые 1–3 часа.
Эта группа формируется **строго на основе сигналов аккаунта и контента** — если они указывают на US, тест-когорта будет американской.

Прошёл тест (метрики выше порога) → следующая когорта 2К–10К → снежный ком.
Не прошёл → видео "умирает" в этих 200–500 просмотрах.

### Иерархия GEO-сигналов

| Приоритет | Сигнал | Как влияет |
|-----------|--------|-----------|
| 🔴 **#1** | IP в момент публикации | Устанавливает географию первой когорты |
| 🔴 **#2** | SIM-карта аккаунта | Устанавливает "домашнее" GEO аккаунта |
| 🟡 **#3** | Язык контента (речь + текст) | Дополнительный GEO-сигнал, усиливает или ослабляет |
| 🟡 **#4** | Язык подписей (captions) | Влияет на SEO внутри TikTok + начальное распределение |
| 🟢 **#5** | Хэштеги | Минимальное влияние в 2026 (алгоритм читает контент напрямую) |

**Критическое правило:** если US IP + не-US SIM → конфликт → алгоритм не знает куда выдавать → смешанный охват, не US.
Все три главных сигнала (IP + SIM + язык) должны согласованно говорить "USA".

### Контентные сигналы для US-выдачи

**Речь (spoken audio)**
- TikTok использует ASR (автоматическое распознавание речи) — транскрибирует и индексирует
- Английская речь = сигнал для US распределения
- Американский акцент предпочтительнее британского для US-когорты

**Текст на экране (on-screen text)**
- Взвешивается наравне с spoken audio — важнее, чем подписи
- Английский текст на экране = GEO сигнал
- Ключевые слова в первых 2 секундах видео = приоритет

**Подписи (captions)**
- Ключевые слова в captions дают +20–40% к поиску внутри TikTok
- Пиши captions на английском, 3–5 ключевых слов
- Caption ≠ хэштег: пиши предложениями, не через #

**Хэштеги** (2026)
- Алгоритм анализирует контент напрямую через CV + NLP
- Хэштеги полезны для TikTok Search, но не влияют на алгоритмическую дистрибуцию
- 3–5 хэштегов максимум: 2 широких (english), 1 нишевый

### Метрики, которые двигают видео дальше

| Метрика | Вес в алгоритме | Порог для US виральности |
|---------|----------------|--------------------------|
| Watch time / Completion rate | ~40–50% | **≥70%** (в 2025 подняли с 50%) |
| Rewatch rate | очень высокий | любой позитив |
| Saves | высокий | >2% от просмотров |
| Shares | высокий | >1% от просмотров |
| Comments | выше лайков | любые — особенно дискуссии |
| Likes | низкий | не главное |

**Главное правило 2026:** 60-секундное видео с 80% completion > 15-секундное с 95% completion.
Алгоритм смотрит на **watch-time per impression**, а не на процент досмотра.

### Технология контента для US GEO (практика)

**Хук (первые 3 секунды)**
- Pattern interrupt: цифра / шок / вопрос / контринтуитивное заявление
- На экране: крупный текст на английском с ключевым словом
- Речь: первая фраза должна заставить НЕ листать дальше

**Структура видео для completion ≥70%**
```
0–3 сек:   Pattern interrupt (хук)
3–15 сек:  Проблема + почему ты должен досмотреть
15–45 сек: Трансформация / решение с доказательством
45–55 сек: Climax (главное обещание выполнено)
55–60 сек: CTA — мягкий, без "купи сейчас"
```

**Звук (audio)**
- Использовать трендовые US-аудио → получаешь хвост чужой аудитории
- Нишевый аудио-трек, не трендовый → алгоритм ставит в профильную нишу
- Оба варианта рабочие — зависит от стратегии

**Частота публикаций**
- Прогрев (дни 1–7): 1–2 видео/день
- Активная фаза: 2–3 видео/день максимум
- Между постами: минимум 4 часа (HumanJitter)
- Консистентность важнее количества — пропуск 3+ дней сбрасывает траст

### Nutra-специфика для US

| Нельзя | Можно |
|--------|-------|
| "cures diabetes" | "supports blood sugar balance" |
| "lose 10 lbs in a week" | "my 30-day journey" |
| Таблетки крупным планом | Упаковка продукта |
| "FDA approved" | "made in USA" |
| До/после в медицинском контексте | Lifestyle before/after |

Автофильтры TikTok блокируют: weight loss + конкретные сроки, acne/skin claims, hormone content.

### Betting-специфика для US

- Poker strategy / sports analysis работает, явные призывы к ставкам → бан
- Аудитория: мужчины 20–40 лет
- Формат: "Как я анализирую матчи" (образовательный, не рекламный)
- CTA без слов "bet", "gambling" — "link in bio" или свайп

### Применение для UBT OS

1. **A21 content_creator** → промпты уже содержат структуру хука + brand voice per GEO
2. **A19 text_humanizer** → Stop-Slop убирает AI-маркеры, которые снижают completion rate
3. **A25 compliance_gate** → блокирует nutra/betting нарушения до публикации
4. **A13 HumanJitter** → минимум 4ч между постами для antiban
5. **Новое:** учитывать on-screen text как GEO-сигнал — A21 должен включать EN текст-оверлеи в сценарий
6. **Новое:** target completion rate ≥70% → videos ≤60 сек с retention-паузами (A23 уже это делает)
