# Алгоритмы GEO-таргетинга для выхода на US-аудиторию

## Контекст
Материал из курса "Зарубежка" — практика работы с иностранной аудиторией (фокус на США).
Применяется к вертикалям **betting / nutra** на GEO **US, BR, MX, DE, PL**.

## Алгоритм 2 — BEHAVIOR (Поведение) / Технический прогрев

### Временное окно
Первые **3–4 дня** владения аккаунтом определяют всё:
- Плато 200–500 просмотров (участь большинства) — если прогрев не сделан
- Зелёный свет: первые ролики на 5–10К просмотров — если всё правильно

### Что такое технический прогрев
Цель — доказать алгоритму два факта одновременно:
1. Ты = живой человек (не бот, не робот, не мошенник)
2. Ты = человек из Америки (не смотря на реальную географию)

### Что входит в прогрев
**Верхушка айсберга (видимое):**
- Лайки, репосты, комментарии, настройка рекомендательной ленты

**Технический уровень (скрытое):**
- Маскировка IP-адресов + привязка к конкретному существующему US-адресу
- Получение billing-адреса и платёжных реквизитов (как у настоящего американца)
- Авторские тонкости (9-урочный модуль в курсе)

### Ключевой принцип
Нельзя начинать постить контент, пока не доказана "человечность" + US-география.
Контент до прогрева = рулетка.

### Применение для UBT OS
- **Blotato-автопостинг** = паттерн бота → нужен слой human-behavior simulation перед публикацией
- IP для аккаунтов: не просто US-прокси, а привязка к конкретному US-адресу (улица, город)
- Нужен billing address + платёжные данные для верификации US-личности аккаунта
- Warmup-период (3–4 дня) должен быть зашит в логику запуска каждого нового аккаунта
- В курсе есть ссылки на "секретные сервисы" для этого — уточнить

## Алгоритм 1 — ORIGIN (Происхождение)

### Как работает
Платформа считывает уникальный цифровой след устройства — аналог IMEI/device fingerprint.
Это происходит **до** любой оценки контента.

### Типы device ID
| ID | Что означает | Влияние |
|---|---|---|
| `RU` | Российское устройство | Охват режется на старте |
| `CN` | Китайское устройство | Охват режется на старте |
| `USA` | Американское устройство | Максимальное преимущество, сложнее добыть |
| `GLOBAL` | Нет привязки к стране | Нейтральный старт — оптимальный выбор |

### Ключевые цифры
- Правильный device = **~20% от итогового успеха**
- Без этих 20% остальные 80% не работают

### Рекомендация из курса
Выбор в пользу `GLOBAL` — нейтральный идентификатор, не требует "американского" устройства, доступен без поездок за рубеж.

## Применение для UBT OS

### Проблема
Серверы и устройства с `RU` device fingerprint режут органический охват ещё до публикации контента. При автопостинге через **Blotato** это означает, что аккаунты могут системно недополучать US/EU трафик.

### Что проверить
- [ ] С каких устройств/серверов идёт активность аккаунтов в Blotato
- [ ] Какой device fingerprint у используемых эмуляторов Android/iOS
- [ ] IP-адреса: нужны не просто US-прокси, а связка IP + device ID в одной "стране"

### Что делать
1. **Устройства:** использовать телефоны/эмуляторы с `GLOBAL` или `US` device ID
2. **Прокси:** мобильные US-прокси (не дата-центр — легко детектятся) в паре с нужным device ID
3. **Уточнить:** поставщик "глобальных" устройств упомянут в курсе — получить контакт

### Приоритет
`CRITICAL` — это фундамент. Без правильного device ID контент-стратегия не даст US-охвата.
